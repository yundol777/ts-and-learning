## 학습 로그 #1

**시간**: 03/24 03:00 ~ 04:30 (약 \\90분)
**학습 범위**: typescript 공식문서

### 1. 첫 학습 전략 설계

AI를 활용해 공식문서를 빠르게 읽으며 흐름을 파악한다.

- 그 전략을 선택한 이유는?
  몇 번 사용해보기는 했지만, 공식문서를 한번도 읽어보지 않았다.
  왜 이 기술이 나왔고, 어떤 식의 사용을 권장하는지 알아보고 싶었다.

- 실제로 어떻게 학습했는지 디테일한 과정을 써보세요.

1. 문제와 목적을 찾는다.
   (왜 이 기술이 나왔고, 어떤 문제를 해결하기 위해 나왔는지)

2. AI를 활용해 공식문서를 빠르게 읽으며 흐름을 파악한다.

3. 공식문서를 어느정도 소화한 후 문제를 풀며 추가적으로 학습한다.

### 2. 전략 평가

- 효과적이었던 것과 그 이유
  기술이 어떤 목적으로 나왔고, 어떤 것을 해결할 수 있는지에 대해 확실히 이해했다.
  기본적인 사용법을 익혔다.

- 비효과적이었던 것과 그 이유
  사실 공식문서를 처음부터 끝까지 읽는 것은 조금 효율이 떨어지는 느낌을 받았다.
  기존 학습과 동일한 이유이다. 특히 핸드북은 단순히 어떤 상황에 사용할 수 있는 기술들의 나열로, 읽기 너무 어려웠다.
  억지로 읽어도 남는게 별로 없었다. 생각해보면 학습법에서 내가 이걸 왜 공부해야하는지에 대한 문제 정의가 부족했던 것 같다.
  어떤 문제를 해결하는게 아닌 단순히 공식 문서부터 읽다보니, 문제와 목적을 찾고 나니 어디에 목표를 잡고 읽어야 하는지가 모호해졌다.

### 3. AI 피드백

- 자신의 학습 전략에 대해 AI에게 피드백을 요청하고, 유용했던 제안 1가지 이상 기록
  “문서를 먼저 읽지 말고, 만들다가 막힐 때 필요한 만큼만 읽어라.”

### 4. 다음 타임에 바꿀 것

- 유지할 것과 그 이유
  공식문서를 참고하는 것은 유지할 것 같다. 두꺼운 책 보다는 작은 단위로 공식문서가 나눠져있어서, 빠르게 찾아 읽기 좋은 것 같다.

- 바꿀 것과 그 이유
  내가 생각한 내용과 AI 피드백이 거의 일치했다.
  이해가 안가는 부분의 문서를 다시 읽기 보다는 일단 미션인 typescript-exercises를 알고있는 지식 내에서 풀어볼 것 같다.
  그리고 모르는 부분이 나왔을 때 해당 키워드를 뽑아 공식문서 핸드북을 읽는 방식으로 변경하여 공부를 진행해볼 예정이다.

## 학습 로그 #2

**시간**: 03/24 01:00 ~ 07:00 (약 360분)
**학습 범위**: typescript-exercises #1~#9

### 1. 이번 타임의 학습 전략

- 이전에 바꾸기로 한 전략은 무엇이었고, 실행했는가?
  공식문서를 충분히 소화하는 것을 포기하고, 문제부터 풀어보며 학습하는 전략을 세웠고, 실행했다.

- 실제로 어떻게 학습했는지 디테일한 과정을 써보세요.

1. 문제를 먼저 읽고 풀어본다.
   1-1. 문제가 풀리면 7번으로.

2. 문제가 풀리지 않으면 하단에 기재된 관련 공식 문서 링크로 들어가 내용을 학습한다. (문제 해결을 최우선으로 생각하며 필요한 내용을 찾아 공부한다.)

3. 학습 후 문제를 다시 풀어본다.
   3-1. 문제가 풀리면 7번으로

4. 그래도 풀리지 않으면 AI에게 생각의 흐름과 문제를 설명하며 부족한 부분을 키워드로 제공받는다.

5. 해당 키워드를 방법을 가리지 않고 추가적으로 학습한다.

6. 학습 후 문제를 다시 풀어본다.

7. 문제가 풀리면 AI에게 피드백을 요청한다. (피드백은 답안이 아닌 키워드로 제공받는다.)

8. 피드백을 기반으로 더 좋은 답변을 고민해보고 피드백을 재요청한다.

9. 답변 일치 응답을 받으면, 문제를 풀며 학습하거나 새롭게 알게 된 내용을 도식화하여 정리한다.

10. 다음 문제를 푼다.

### 2. 전략 평가

- 효과적이었던 것과 그 이유
  우선 자료의 난이도가 적절해서 좋았다.
  문제를 1번부터 풀어가면서 학습이 자연스럽게 이어졌고, 너무 쉽지도 어렵지도 않아 흐름이 끊기지 않았다.

특히 혼자 고민하는 시간이 늘어나면서 학습의 깊이가 달라졌다고 느꼈다.
공식문서를 단순히 읽을 때보다, 풀리지 않는 문제를 해결하기 위해 필요한 내용을 찾아보는 과정에서 훨씬 더 몰입할 수 있었다.

또한 부족한 부분이 명확해지면서 학습의 목적이 분명해졌고, 그에 따라 공부해야 할 범위도 자연스럽게 줄어들었다.
문제 단위로 풀이와 기록을 남기다 보니 기록 자체에 대한 부담도 크지 않았다.

마지막으로, 단순히 이해가 가지 않는 부분을 설명받는 것이 아니라
문제를 푸는 과정에서 나의 사고 흐름을 피드백 받고 바로 수정할 수 있다는 점이 특히 인상적이었다.

- 비효과적이었던 것과 그 이유
  사실 이전에 비해 모든 부분이 효과적이었던 것 같다. 오히려 너무 몰입해서 문제를 고민하고 풀어보는 시간이 생각보다 많이 늘어난 것 같다.
  혼자 고민하는 시간은 분명히 필요하지만, 너무 길어지면 효율이 떨어지는 느낌을 받았다.
  명확한 시간을 정해두고 다음 과정으로 넘어가는 전략이 추가되면 좋을 것 같다고 생각했다.

### 3. AI 피드백

- 자신의 학습 전략에 대해 AI에게 피드백을 요청하고, 유용했던 제안 1가지 이상 기록
  "문제 해결 시 고민 시간을 10~15분으로 제한하고, 그 이후에는 힌트나 키워드를 활용해 다음 단계로 넘어가라."

### 4. 다음 타임에 바꿀 것

- 유지할 것과 그 이유
  남은 문제를 비슷한 학습법을 유지하며 풀어볼 것 같다.
  학습하면서도 해당 방법이 나에게도, 객관적으로도 좋은 방법이라고 생각했다.

- 바꿀 것과 그 이유
  고민하는 시간을 추가해볼 것 같다.
  그리고 지금은 AI를 활용할 때 매번 프롬프트에 단계별로 요구사항을 적고 있는데, 이를 좀 더 편하게 사용할 수 있는 도구를 만들어 볼 예정이다.

## 학습 로그 #3

**시간**: 03/25 09:30 ~ 11:30 (약 120분)
**학습 범위**: typescript-exercises #10

### 1. 이번 타임의 학습 전략

- 이전에 바꾸기로 한 전략은 무엇이었고, 실행했는가?
  AI를 이용할 때 도구를 만들어서 사용해보았다.

- 실제로 어떻게 학습했는지 디테일한 과정을 써보세요.
  과정은 이전과 동일하고, AI를 사용할 때 만들어둔 도구를 이용했다.
  문제 번역, 키워드, 정답 피드백, 모범 답안. 이렇게 총 4가지 단계로 도구를 설계하여 각 학습 단계에서 적절하게 사용하였다.

### 2. 전략 평가

- 효과적이었던 것과 그 이유
  매번 프롬프팅을 추가적으로 하는 부분이 줄어들었다. 사고 흐름을 물어보거나, 키워드를 요구하고 싶을 때 항상 문제 넣고, 정답은 말하지 말고 키워드만 뽑아달라고 요청을 보냈었는데, 항상 답변 형식도 다르고 매번 같은 요청을 반복적으로 하는 느낌이 들었었다. 그래서 도구를 설정하고 각 단계 별로 구분하여 설정해두니, 처음에 문제만 보내면 번역하여 깔끔하게 정리해주고, "키워드" 글자만 치면 해당하는 키워드가 나오는 식으로 프롬프팅 시간을 줄여 효율적으로 사용할 수 있었던 것 같다.

- 비효과적이었던 것과 그 이유
  10번 부터는 난이도가 조금 어려웠었는데 키워드만 가지고 고민하는 시간이 너무 길었던 것 같다.
  키워드를 공부해도 문제 상황이 복잡해서 쉽게 풀리지 않았고, 시간을 정해서 반복했지만 키워드만으로는 한계가 있었던 것 같다.

### 3. AI 피드백

- 자신의 학습 전략에 대해 AI에게 피드백을 요청하고, 유용했던 제안 1가지 이상 기록
  키워드로 방향 잡고 → 힌트로 확장하고 → 부분정답으로 막힌 지점만 뚫은 뒤 → 반드시 스스로 다시 구현한다.

### 4. 다음 타임에 바꿀 것

- 유지할 것과 그 이유
  전체적인 방향성은 유지할 것 같다.

- 바꿀 것과 그 이유
  키워드를 받고 일정 시간이 지나면, 힌트, 부분 정답 식으로 점진적으로 풀이를 제공받을 것 같다.
  키워드만 가지고는 문제 해결에 도움을 받는데에 조금 한계를 느꼈다.
  조금 더 효율적인 풀이를 위해 단계별로 도움을 받은 뒤 다시 풀어보는 단계를 추가할 것 같다.
